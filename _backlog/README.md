# Backlog Drafts

Use this folder for raw ideas, notes, pasted outlines, rough technical findings,
links, transcripts, or unfinished drafts.

Files in `_backlog` are **not published** by the blog. The app only reads
Markdown files from `_posts`.

## Suggested Filename

```text
topic-short-name.md
```

or, when the date matters:

```text
YYYY-MM-DD-topic-short-name.md
```

## Suggested Shape

```markdown
# Working title

## Goal

What should the final post teach or explain?

## Audience

Who is this for?

## Raw notes

- Paste rough bullets here.
- Add links, code snippets, screenshots to describe, or command outputs.
- Do not worry about final writing quality.

## Must include

- Point one
- Point two

## Avoid

- Things that should not go into the final post.
```

When asking Claude/Codex to publish a backlog item, reference the file and ask it
to turn the raw material into a post.
